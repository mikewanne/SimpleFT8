## Neue Befunde

**KP1 — start_qso Reset auch fuer CQ_WAIT** ⛔ KRITISCH
Aktuell `if state not in (IDLE, CQ_WAIT):` → CQ_WAIT skippt Reset.
Aber bei CQ_WAIT kann `_pending_reply` gesetzt sein → bleibt bei
Stations-Wechsel haengen, triggert spaeter unerwartet.
**Fix:** Bedingung auf `if state != IDLE:` aendern, plus Pendings
explizit resetten.

**KP2 — Queue-Eintrag bei Klick auf bereits gewartete Station nicht entfernt**
Wenn Caller in `_caller_queue` steht und User klickt manuell auf ihn
→ Caller wird beim spaeteren Queue-Abarbeiten nochmal kontaktiert.
**Fix:** in `_on_station_clicked` Queue um msg.caller bereinigen:
```python
self.qso_sm._caller_queue = [
    m for m in self.qso_sm._caller_queue if m.caller != msg.caller
]
self.qso_sm.queue_changed.emit([m.caller for m in self.qso_sm._caller_queue])
```

**KP3 — _active_qso_targets wachsende Set bei Stations-Wechsel**
Z.84 fuegt nur neue their_call hinzu. Alte their_call bleibt 150s.
Bei vielen schnellen Wechseln → Set waechst unnoetig.
**Fix:** alte their_call vor add() discarden:
```python
old_call = self.qso_sm.qso.their_call if self.qso_sm.qso else None
if old_call:
    self._active_qso_targets.discard(old_call)
self._active_qso_targets.add(msg.caller)
```

Behoben durch KP1.
