## 1. Bug-Wurzeln (final)

### 🔴 W1: `start_qso` resetet keine Pendings bei state != IDLE
**Datei:Zeile:** `core/qso_state.py:241-245`
**Aktuell:**
```python
if self.state not in (QSOState.IDLE, QSOState.CQ_WAIT):
    self._set_state(QSOState.IDLE)
```
**Problem:** `_pending_reply`, `_pending_hunt_reply`, `_pending_rr73`
bleiben gesetzt mit alter their_call. Plus: CQ_WAIT wird komplett
geskippt (Pendings koennen dort gesetzt sein).

### 🔴 W2: `_caller_queue` enthaelt manuell-gewaehlte Station
**Datei:Zeile:** `ui/mw_qso.py:_on_station_clicked` Z.84
**Problem:** Wenn angeklickte Station bereits in Queue → wird beim
Resume nochmal kontaktiert (Doppel-QSO-Risiko, Folge-ADIF-Eintrag).

### 🟡 W3: `_active_qso_targets` waechst monoton bei Wechseln
**Datei:Zeile:** `ui/mw_qso.py:_on_station_clicked` Z.84
**Problem:** Alte their_call bleibt 150s im Set (Decoder behaelt
Stations-Daten unnoetig lange).

### 🟡 W4: `_was_cq` State-Machine-extern korrigiert
**Datei:Zeile:** `ui/mw_qso.py:_on_station_clicked` Z.100-103
**Problem:** Workaround macht State-Machine-Verantwortung
inkonsistent. Robust nur wenn extern korrekt aufgerufen.

### 🟡 W5: TX-Klick silent ignoriert
**Datei:Zeile:** `ui/mw_qso.py:_on_station_clicked` Z.68-70
**Problem:** Mike klickt waehrend TX, sieht nichts. UX-Frust.

### 🟢 W6: Auto-Hunt-Resume unklar bei Cancel/HALT
**Datei:Zeile:** `ui/mw_qso.py:_on_cancel`, `_on_station_clicked`
**Problem:** wenn manueller QSO durch HALT/zweiten Klick beendet wird,
Auto-Hunt bleibt evtl. dauerhaft pausiert.
**To-Verify:** `core/auto_hunt.py` API (im Plan-V1).
