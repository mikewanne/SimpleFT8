## 1. Ziel

1 Gain-Messung (**8 Zyklen** Rohdaten, ~2 Min) → **2 Auswertungen** parallel
(Standard zählt Stationen, DX zählt SNR) → **beide Stores** atomar
gespeichert. Modus-Wechsel Std↔DX wird instant (Gain-Validität 6h gilt
für beide Stores aus 1 Messung).

Korrigiert gegenüber V2: nicht „18 Zyklen" — der Code hat
`ROUNDS=2 × 2 Antennen × 2 Gain-Stufen = 8` (Code-Verifikation R1-F1).
Ersparnis: heute 2 Mess-Sessions à 8 Zyklen = 16. Künftig: 1 × 8 = **50%
gespart**.

## 2. Akzeptanzkriterien

**AC1 — DXTuneDialog `_finish()` produziert beide Auswertungen (R1-F1):**
- Neuer Helper:
  ```python
  def _best_for(self, ant: str, use_snr: bool) -> dict:
      """Liefert {gain, avg, count} für die scoring-Variante.

      use_snr=True  → DX-Optimum (bestes Top-5-SNR pro Gain-Stufe).
      use_snr=False → Standard-Optimum (meiste Stationen pro Gain-Stufe).
      """
  ```
- `_finish` ruft `_best_for(ant, False)` (Std) und `_best_for(ant, True)`
  (DX), baut `_results`:
  ```python
  self._results = {
      "standard": {
          "ant1_gain": int, "ant2_gain": int,
          "ant1_avg": float, "ant2_avg": float,
          "best_ant": str, "best_gain": int,
      },
      "dx": { gleiche Keys },
      # Top-Level = Kopie aktiver Modus (Backwards-Compat R1-F8):
      "ant1_gain": ..., "ant2_gain": ..., "ant1_avg": ..., "ant2_avg": ...,
      "best_ant": ..., "best_gain": ...,
  }
  ```
- Top-Level = `_results["dx"]` wenn `scoring_mode=="snr"`, sonst
  `_results["standard"]`. Docstring von `_finish` und `get_results`
  dokumentiert die bewusste Doppel-Struktur (R1-F8).
- `set_rx_antenna(ant)` / `set_rfgain(gain)` Z.398-399 nutzen weiter die
  Top-Level-Werte (= aktiver Modus am Radio anwenden).

**AC2 — mw_radio schreibt beide Stores, defensiver Fallback (R1-F4):**
- In `ui/mw_radio.py:1438+`:

  if has_dual:

      gain_mode = getattr(self, '_gain_scoring_mode', 'snr')
      store = (self._dx_store if gain_mode == "snr" else self._standard_store)
      if store:
          store.save_gain(band, ft_mode, ...)

## 4. Randbedingungen

- **PySide6 Signal/Slot**.
- **Hardware ANT1=TX:** `_finish` Z.400 `set_tx_antenna("ANT1")` bleibt.
- **Thread-Safety:** GUI-thread only.
- **PresetStore atomar:** `save_gain` returnt False bei Disk-Fehler →
  WARN-Log statt Crash (R1-F4-Defensive).
- **Reversibilität:** Bei Disk-Fehler ist 1 Store evtl. veraltet. Selten.
  Field-Test zeigt.
- **Adaptive-Stop OR-Logik:** bewusst gewählt (R1-F2 Doku).
- **`_gain_scoring_mode` unangetastet:** Trigger-Saver bleibt.
- **`settings.save_dx_preset` als Code-Sterbe-Pfad:** wird in P51 nicht
  mehr gerufen, Funktion existiert weiter für Cleanup-Workflow.
