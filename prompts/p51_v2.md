## 1. Ziel

1 Gain-Messung (18 Zyklen Rohdaten) → **2 Auswertungen** (Standard zählt
Stationen, DX zählt SNR) → **beide Stores** atomar gespeichert.
Modus-Wechsel Std↔DX wird damit instant (kein 90-Sek-Neu-Mess-Block).

Hintergrund: `_phase_data` in `ui/dx_tune_dialog.py` ist
scoring-unabhängig — beide Auswertungen sind aus denselben Rohdaten
ableitbar. Mike beobachtete 14.05.: 20m FT8 hat Std 10/10 vs DX 20/10
weil zu unterschiedlichen Zeitpunkten gemessen. P51 macht das konsistent.

## 2. Akzeptanzkriterien

**AC1 — DXTuneDialog `_finish()` produziert beide Auswertungen:**
- Neuer Helper in `ui/dx_tune_dialog.py`:
  ```python
  def _best_for(self, ant: str, use_snr: bool) -> dict:
      """Liefert {gain, avg, count} für die scoring-Variante."""
  ```
- `_finish` ruft `_best_for(ant, use_snr=False)` für Standard +
  `_best_for(ant, use_snr=True)` für DX, baut beide Sätze:
  ```python
  self._results = {
      "standard": {
          "ant1_gain": int, "ant2_gain": int,
          "ant1_avg": float, "ant2_avg": float,
          "best_ant": "ANT1"|"ANT2", "best_gain": int,
      },
      "dx": { gleiche Keys },
      # Top-Level (Backwards-Compat) — spiegelt aktiven scoring_mode:
      "ant1_gain": ..., "ant2_gain": ..., "ant1_avg": ..., "ant2_avg": ...,
      "best_ant": ..., "best_gain": ...,
  }
  ```
- Top-Level-Felder = Kopie aus `_results["standard"]` wenn `scoring_mode
  == "stations"`, sonst aus `_results["dx"]`. Bestehender Aufrufer-Code
  bleibt funktional.
- `set_rx_antenna(ant)` / `set_rfgain(gain)` Z.398-399 in `_finish` nutzen
  weiter den aktiven Modus (Top-Level-Werte) — Radio bekommt den Satz für
  den der User gerade misst.

**AC2 — mw_radio `_on_dx_tune_accepted` schreibt beide Stores:**
- In `ui/mw_radio.py:1438+`:
  ```python
  r = dialog.get_results()
  std_data = r.get("standard", r)  # Fallback fuer Dialog ohne P51
  dx_data  = r.get("dx", r)
  std_ok = self._standard_store.save_gain(
      band, ft_mode,
      rxant=std_data.get("best_ant", "ANT1"),
      ant1_gain=std_data.get("ant1_gain", 0),
      ant2_gain=std_data.get("ant2_gain", 0),
      ant1_avg=std_data.get("ant1_avg", 0.0),
      ant2_avg=std_data.get("ant2_avg", 0.0),
  ) if self._standard_store else False
  dx_ok = self._dx_store.save_gain(...) if self._dx_store else False
  if not std_ok or not dx_ok:
      print(f"[P51] WARN: save_gain returned False — std_ok={std_ok}, dx_ok={dx_ok}")
  ```
- Beide Calls **unabhängig** — wenn EIN Store fehlschlägt (Disk-Fehler,
  return False) läuft der andere trotzdem (Try-Except um beide
  Save-Calls).
- `set_rfgain` am Radio nutzt weiter den aktiven Modus
  (`_gain_scoring_mode` via Top-Level-Werte).

**AC3 — `settings.save_dx_preset` zweimal:**
- `_on_dx_tune_accepted` Z.1458 wird zu zwei Aufrufen:
  - `save_dx_preset(scoring="standard", ant1_gain=std_data["ant1_gain"], …)`
  - `save_dx_preset(scoring="dx", ant1_gain=dx_data["ant1_gain"], …)`
- Beide schreiben in `settings.json` unter `dx_preset_{band}_{mode}_{scoring}`
  (bestehendes Schema). Kein Daten-Verlust, kein neues Migration-Bedarf.

**AC4 — Display zeigt beide Optima:**
- `_update_results_display` markiert pro `(ant, gain)`-Kombination ob es
  Standard-Optimum, DX-Optimum oder beides ist:
  - Std+DX-gleicher Gain: `←(Std+DX)`
  - Nur Std: `←(Std)`
  - Nur DX: `←(DX)`
- `step_label` während Messung: „Messung läuft (gilt für beide Modi)".
- `scoring_text` am Ende: „Bewertet nach SNR (DX) UND Stationsanzahl
  (Standard)" — zeigt beide.

**AC5 — Mode-Wechsel mid-session ist instant:**
- Nach erfolgreicher P51-Messung sind beide Stores valid (gleicher
  `gain_timestamp`). Bestehende Logik im Mode-Wechsel
  (`_check_diversity_preset` / `_open_dx_tune_dialog`) triggert keine
  neue Messung wenn der Ziel-Store valid ist — das ändert sich nicht
  durch P51, ist aber jetzt EFFEKTIV durch Doppel-Save.
- Field-Test F1 prüft das praktisch.

**AC6 — `_gain_scoring_mode` bleibt für UI-Anzeige/Anwendung:**
- Attribut NICHT entfernen. Toggle bleibt im Dialog (oder Settings).
- Toggle bestimmt nur welcher Werte-Satz am Radio aktiv angewendet wird,
  NICHT mehr ob/wie gemessen wird.

**AC7 — Backwards-Compat:**
- Falls neuer `_results`-Code aus älterem Dialog kommt (z.B. Tests die
  noch alten Single-Dict-Stil zurückgeben), `_on_dx_tune_accepted`
  benutzt `r.get("standard", r)` als Fallback. Kein Crash.

**AC8 — Adaptive-Stop unverändert:**
- `_check_adaptive_stop` Z.270-318: prüft weiter nur den aktiven
  scoring-Modus (Δ_SNR ≥ 4 dB ODER Δ_STAT ≥ 50%). KISS. Wenn aktive
  Auswertung früh klar ist, stop. Die andere Auswertung kriegt im worst
  case knappere Daten, aber Mindest-5-Stationen + 4-Bucket-Check
  schützen.
- R1-Frage offen: ist OR-Logik (stop wenn EIN Modus klar) korrekt für
  P51?

## 4. Randbedingungen

- **PySide6 Signal/Slot**.
- **Hardware ANT1=TX:** `_finish` Z.400 `set_tx_antenna("ANT1")` bleibt.
- **Thread-Safety:** GUI-thread only (cycle_decoded-Signal-Slot in
  GUI-thread). Keine neuen Locks.
- **PresetStore atomarisch:** `save_gain` returnt False bei Disk-Fehler.
- **Adaptive-Stop:** OR-Logik unverändert (siehe AC8, R1-Frage offen).
- **`_gain_scoring_mode`:** bleibt, Toggle bleibt.
- **Reversibilität:** Wenn Store-1-save erfolgreich + Store-2-save
  fehlschlägt, sind beide Stores unabhängig valid mit eigenen
  Timestamps. Inkonsistenz möglich aber selten — Field-Test zeigt.
