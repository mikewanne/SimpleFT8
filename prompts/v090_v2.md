## Problem (KRITISCH 🔴, R1-Audit 2026-05-04 + Code-Verifikation)

`core/diversity.py:84-86`:

```python
if self._phase == "measure":
    # Messung: A2,A1,A1,A2 (6-Slot nahtlos, beide Paritaeten)
    return ("A2","A1","A1","A2","A1","A1")[self._measure_step % 6]
```

Pattern enthaelt:
- Slot 0: A2  | Slot 1: A1  | Slot 2: A1
- Slot 3: A2  | Slot 4: A1  | Slot 5: A1

= **4× A1 + 2× A2** — identisch mit OPERATE 70:30. Doku Z.19 sagt
„MESS-PHASE (6 Zyklen): 3×A1 + 3×A2 messen". Code lügt.

**Wirkung:** ANT2 strukturell unter-gemessen → Median-Vergleich verschoben →
8 %-Schwelle bevorzugt ANT1-Ratio → 70:30 ANT1 dominant → Mike's Beobachtung
4 % ANT2-Win-Rate auf 40 m FT8 wird teilweise davon erklaert.

**Statistik-Disclaimer:** Alle Pre-v0.90 Diversity-Daten haben strukturellen
Mess-Bias 4:2 statt 3:3. Pooled-Mean +88 %/+124 % bleibt valide weil ANT2
trotz Bias signifikant beitraegt — absolute ANT2-Win-Rate ist konservativ
unterschaetzt. Faire Messung ab v0.90 wird wahrscheinlich hoehere Win-Rates
zeigen.

## Fix-Entscheidung (Plan-Datei „Empfehlung Default-V1": Option C)

```python
return ("A1","A1","A2","A2","A1","A2")[self._measure_step % 6]
```

- 3× A1 + 3× A2 → fair ✅
- A1 hat zusammenhaengendes Even+Odd-Paar (Slots 0-1)
- A2 hat zusammenhaengendes Even+Odd-Paar (Slots 2-3)
- Singletons 4 (A1=even) + 5 (A2=odd) → leichte Asymmetrie in Singletons
  (akzeptabel, beide Antennen sehen mind. ein zusammenhaengendes Paar)
- 6 Slots bleiben → MEASURE_CYCLES=6 unveraendert → Pipeline-Kosten **null**
- Loop-Uebergang: Pos 5 (A2) → Pos 0 (A1) — kein 3× hintereinander ✓
- Max consecutive: 2 (A1A1 oder A2A2) ✓

**Verworfene Alternativen:**
- Option A `("A1","A2","A1","A2","A1","A2")` — fair aber Even/Odd-Bias
  (jede Antenne nur 1 Paritaet)
- Option B `("A1","A1","A2","A2","A1","A1","A2","A2")` mit MEASURE_CYCLES=8 —
  rollt Block 1 #4 (-30 s Pipeline) zurueck
